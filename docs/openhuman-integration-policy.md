# OpenHuman Reference Policy

AI Development OS may use OpenHuman only as architectural inspiration for retrieval-first memory, hierarchical summaries, local-first summarization, context compression, provider abstraction, memory tree design, and long-term continuity scaling.

This repository does not import OpenHuman runtime authority, autonomous agent orchestration, uncontrolled RAG behavior, always-on ingestion, prompt-owned identity systems, or unbounded memory expansion.

## Adopted Ideas

- Hierarchical summaries for large-context reduction.
- Bounded memory tree traversal for continuity retrieval.
- Local-first summarization before expensive model use.
- Context compaction before prompt routing.
- Provider abstraction without provider-owned orchestration.
- Continuity retrieval through summaries rather than raw replay.

## Rejected Ideas

- Autonomous memory ingestion.
- Full repository injection.
- Prompt-owned identity or personality mutation.
- Continuous background reasoning.
- Uncontrolled RAG chatbot behavior.
- Memory-owned orchestration.

## Governance Boundary

Retrieval remains subordinate to AI Development OS governance. Budget pressure, GPT-5.5 suppression, council throttling, diff-only enforcement, and runtime audit take precedence over retrieval expansion.
