# Known Failure Baseline

Known Failure Baseline separates expected baseline failures from new failures. It is a classification runtime, not a suppression runtime.

## Classes

- Known CI failures.
- Known Unity license failures.
- Known lint debt.
- Known formatting debt.
- Known vendor exclusions.
- Acknowledged temporary failures.

## Output

- Baseline failures.
- New failures.
- Unresolved failures.
- Ignored failures.

The runtime keeps failures visible so sprint close and remote verification can decide whether new action is needed.
