# Topic Isolation

Topic Isolation separates architecture, governance, rollout, provider, renderer, experimental, and CI-debt topics from routine sprint sessions.

## Outputs

- Isolated topics.
- Active topics.
- Deferred topics.
- Fork session recommendation.
- Architecture session recommendation.

This runtime isolates topics without fragmenting every small task. Routine sessions can retain CI debt context while deferring architecture and provider migration discussions.
